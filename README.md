### 🔍 Ответы на вопросы анализа
| Вопрос | Ответ |
|--------|-------|
| **Какие признаки использованы и почему?** | `median_income`, `longitude`, `latitude`, `housing_median_age`, `ocean_proximity`. Оставлены признаки с высокой корреляцией с целевой переменной и логической значимостью для стоимости жилья. |

## Задача 2: DVC-пайплайн
Построен воспроизводимый пайплайн в файле `dvc.yaml`. Этапы:
- `prepare` → загрузка, очистка, масштабирование, OHE, логарифмирование target
- `train_linear` → линейная регрессия
- `train_decision_tree` → дерево решений
- `train_catboost` → CatBoost
- `train_xgboost` → XGBoost
- `train_mlp` → многослойный перцептрон

При изменении `params.yaml` или `data/raw/housing.csv` DVC автоматически пересчитывает зависимые этапы. Конфигурация вынесена в `params.yaml`.

## Анализ результатов и выбор лучшей модели

### Ключевые наблюдения
1. **XGBoost показал наилучший результат** на тестовой выборке (R² = 0.8370), превзойдя остальные модели по всем метрикам.
2. **Малый разрыв между Train и Test** у ансамблевых методов (XGBoost: 0.9195 → 0.8370, CatBoost: 0.8528 → 0.8226) свидетельствует об отсутствии переобучения и хорошей обобщающей способности.
3. **Нейронная сеть (MLP)** показала умеренный результат (R² = 0.7883). Разрыв Train→Test (0.8270 → 0.7883) указывает на лёгкое переобучение, что ожидаемо для небольшого датасета (~19к строк).
4. **Линейная регрессия и дерево решений** показали схожие результаты (~0.64 R²), что подтверждает наличие нелинейных зависимостей в данных, которые эти модели не могут уловить.

### Почему XGBoost победил?
| Фактор | Объяснение |
|--------|------------|
| **Градиентный бустинг** | Последовательное исправление ошибок предыдущих деревьев даёт высокую точность на табличных данных |
| **Устойчивость к масштабу** | Не требует масштабирования признаков, устойчив к выбросам |
| **Регуляризация "из коробки"** | Параметры `subsample`, `colsample_bytree`, `learning_rate` предотвращают переобучение |
| **Работа с нелинейностями** | Эффективно моделирует сложные взаимодействия между географией, доходом и ценой |

### Сравнение с валидационной выборкой
Метрики на Val и Test практически идентичны для всех моделей (разница < 0.02 по R²), что подтверждает:
- Корректность разбиения данных
- Отсутствие "утечки" информации из теста в обучение
- Надёжность валидации как индикатора финального качества
